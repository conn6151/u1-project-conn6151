REQUIREMENTS

If you aim to be a Software Engineer, you will need a certain set of skills and tools available to your disposal. Featured here will be a list of said items that are needed.

SKILLS / KNOWLEDGE

    - Patience: Creating software requires a lot of time and a lot of effort; a majority of programs take a few weeks to make at the least, and at the most, you can expect to wait a few years before you're finished.

    - Multitasking: You'll have multiple things to do while developing pieces of software.

    - Teamwork: Most programs are made among a group of people. You'll need to develop teamwork skills if you want to work efficiently amongst your peers.

SOFTWARE

Recommendations of certain tools will be made; however, I strongly suggest you do your own reasearch into this.
Use what you'd think suit you the best!

Compilier/Interpreter:

    When making software, you'll encounter many moments where the programming language and tools you're using require a specific piece of software before you can actually run what you've made.

    Most times the tools fall into two categories; Compiliers and Interpreters.

    • - Compilier: A program that converts non-executable, human readable code into machine code.
    • Examples include:
      • - GCC/G++ for C/C++
      • - .NET for C#
      • - JDK for Java
    • - Interpreter: A program that goes through human readable code and converts it into machine code as it does that.
    • Examples include:
      • - Web browser for HTML/JS
      • - Python interpreter for... well.. Python
      • - Lua

Operating System:

    It's easy to forget how much the Operating System plays a role into development.

    Developers are actually more likely to use a Linux distribution (distro) when developing their software. This is due to a few benefits the average distro offers:

    • - They give the user full customization and control of the OS; they're "free" as in freedom.
    • - The software developed for Linux is often "open source" (I say often as it isn't guaranteed), meaning that you can look at what code's used to make your OS
    • - You're encouraged to use the command line more often - this may sound like a difficulty, but it increases the speed and efficiency of the developer's work, as long as they know what they are doing

    Meanwhile, the user friendliness of Windows and MacOS is also something that may be nice for some other users. The "it just works" factor is something that a lot of people like. Those Operating Systems are something that everyone's most likely used before, so many may be comfortable using them in the end.

    It all boils to your personal preference. All I suggest for you to do is to try and see which one you like.

    • - Windows 10

    • - Available only on Mac computers.

HARDWARE

    The hardware required is pretty simple. You'll need:

    - A Computer

  • - The CPU should have atleast 4 cores, otherwise the system may feel slow.
  • - The RAM should be atleast 8GB in size, however 16GB is recommended. If you plan on building large things, then 16GB may not even be enough.
  • - Be sure to have a decent amount of IO. Go with about 4 or more USB ports, gen 3.0 or faster (higher number is better)

    • - A Good Chair

  • Software Engineering is a desk job that'll have you sitting sometimes for long periods of time. That's why it's important to choose a good chair; you wouldn't want an aching back all day, would you?

    • - A Monitor

  • - 1080p resolution or above
  • - SDR (HDR if you're developing design software/games)
  • - 600 nits peak brightness or more IF you're in a bright environment (i.e outdoors)
